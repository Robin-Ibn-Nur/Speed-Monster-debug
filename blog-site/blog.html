Storage

We use the website regularly. During the use of the website, some data remains in our browser.

2 types of data are stored in our browser.

  1. Local Storage

    Local Storage permanently stores the data of all the websites we browse in our browser.

  2. Session Storage

    Session Storage stores our single tab's data. After that, the Session Storage becomes empty as soon as the tab is closed. Session Storage is deleted as soon as the tab is closed.

Scope

The scope of a variable defines where it's accessible in a program based on where it was created. Just because we create a variable in one place in our program doesn't mean it's going to be accessible from anywhere in our script.

There are 2 types of variable scope will be disscussed.

  1. Global Scope

    2. The variables defined outside of any function or curly brackets {} are known as global variables and have global scope. Global scope means that the variables can be accessed from any part of that program, any function or conditional state can access that variable.

    Note: It is not a good practice to use global variables when they are not needed as every code block will have access to those variables.

    The keyword var is used to define Global variable

  2. Local Scope

    3. If we have to define some variables inside curly brackets {} or inside a specific function then those variables are called local variables. The local variables have a very confined scope which is called the local scope But, with the release of ES6, the local scope was further broken down into two different scopes:

    • Function scope

      • The function scope is the accessibility of the variables defined inside a function, these variables cannot be accessed from any other function or even outside the function in the main file.

      However, if we try to access this variable outside of the function then we get an error:

    • Block scope.

      • Block scope is also a sub-type of local scope. The block scope can be defined as the scope of the variables inside the curly brackets {}. Now, these curly brackets can be of loops, or conditional statements, or something else. We are only allowed to refer to these variables from within the curly brackets {}

      The keywords let and const are used to define block variables.

JS even Loop

JavaScript is a single-threaded programming language. This means that JavaScript can do only one thing at a single point in time.

The JavaScript engine executes a script from the top of the file and works its way down. It creates the execution contexts, pushes, and pops functions onto and off the call stack in the execution phase.

If a function takes a long time to execute, we cannot interact with the web browser during the function's execution because the page hangs.

function that takes a long time to complete is called a blocking function. Technically, a blocking function blocks all the interactions on the webpage, such as mouse click

Undefine

In JavaScript, three properties, undefined, null, and NAN

The undefined property in JavaScript suggests that a variable has not been assigned a value or not declared at all. The global undefined property describes the primitive value undefined. It is one of JavaScript's primitive data types.

A variable that has not been assigned a value is of type undefined. A method or statement also returns undefined if the variable that is being evaluated does not have an assigned value. A method returns undefined if a value was not returned.

To check if the value is undefined in JavaScript, use the typeof operator. The typeof operator returns a string indicating the type of the unevaluated operand. Use typeof operator with if condition and compare the value of the variable using undefined, and you will get your result

if (typeof data === "undefined") {
console.log("data is undefined")
